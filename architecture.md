# Architecture

## Summary

A single-container FastAPI application serving a REST API (OpenAPI 3.1.0) and a
single-file Vanilla JS GUI. State lives in a SQLite database file mounted as a
Podman volume. The codebase is split into one database module (connection + shared
helpers), three routers (one per entity), three Pydantic model files (one per
entity), a static HTML file, and a main entry point. No ORM is used - raw aiosqlite
SQL keeps the dependency surface minimal and every query fully visible. All side
effects (DB access, current timestamp) are injected, making every component
independently testable.

## Components

### main.py
- Responsibility: assemble the FastAPI application, register lifespan (DB init on
  startup), configure CORS for localhost origins only, mount routers under
  /api/v1, serve static/index.html at GET /.
- Public interface: `create_app() -> FastAPI` (factory function for testability)
- Dependencies: database, routers.org_units, routers.persons, routers.assignments
- FRs covered: FR-29 (lifespan calls init_db), FR-33 (static mount), NFR-08 (CORS),
  NFR-09 (OpenAPI auto-generated by FastAPI)

### database.py
- Responsibility: manage the SQLite connection lifecycle, expose a FastAPI
  dependency `get_db()`, execute schema DDL on startup, and provide two shared
  helper functions used by all three routers.
- Public interface:
  - `init_db(db_path: Path) -> None` - creates all 6 tables if not exist (FR-29, FR-30)
  - `get_db() -> AsyncGenerator[aiosqlite.Connection, None]` - FastAPI dependency
  - `copy_to_history(conn, table: str, row_id: int, now: str) -> None` - copies
    current row to {table}_history with replaced_at = now (FR-32)
  - `active_filter(active_only: bool, today: str) -> str` - returns SQL WHERE
    clause fragment for date range (FR-03, FR-13, FR-22..FR-24)
  - `utcnow_str() -> str` - returns current UTC datetime as ISO string; isolated
    here so tests can monkeypatch it
- Dependencies: aiosqlite, pathlib
- Side effects: filesystem (DB file), all isolated behind injected Path and conn

### routers/org_units.py
- Responsibility: implement all OrgUnit HTTP endpoints.
- Public interface: `router = APIRouter(prefix="/org-units")`
- Endpoints:
  - GET  /              -> list flat  (FR-03, FR-04)
  - GET  /tree          -> nested tree (FR-05)
  - GET  /{id}          -> single record (FR-06)
  - GET  /{id}/history  -> history list (FR-07)
  - POST /              -> create (FR-01, FR-02)
  - PATCH /{id}         -> update with history copy (FR-08, FR-09, FR-10)
  - GET  /{id}/assignments -> assignments for this unit (FR-23)
- Dependencies: database.get_db, database.copy_to_history, database.active_filter,
  database.utcnow_str, models.org_unit
- Pure helper: `build_tree(rows: list[dict]) -> list[dict]` - assembles flat rows
  into nested children structure; pure function, no DB dependency (FR-05)

### routers/persons.py
- Responsibility: implement all Person HTTP endpoints.
- Public interface: `router = APIRouter(prefix="/persons")`
- Endpoints:
  - GET  /              -> list flat (FR-13)
  - GET  /{id}          -> single record (FR-14)
  - GET  /{id}/history  -> history list (FR-15)
  - POST /              -> create (FR-11, FR-12)
  - PATCH /{id}         -> update with history copy (FR-16, FR-17, FR-18)
  - GET  /{id}/assignments -> assignments for this person (FR-24)
- Dependencies: database.get_db, database.copy_to_history, database.active_filter,
  database.utcnow_str, models.person

### routers/assignments.py
- Responsibility: implement all Assignment HTTP endpoints.
- Public interface: `router = APIRouter(prefix="/assignments")`
- Endpoints:
  - GET  /              -> list flat (FR-22)
  - GET  /{id}/history  -> history list (FR-25)
  - POST /              -> create with FK validation (FR-19, FR-20, FR-21)
  - PATCH /{id}         -> update with history copy (FR-26, FR-27, FR-28)
- Dependencies: database.get_db, database.copy_to_history, database.active_filter,
  database.utcnow_str, models.assignment

### models/org_unit.py
- Responsibility: define Pydantic schemas for request validation and response
  serialization for OrgUnit.
- Public interface:
  - `OrgUnitCreate`      - POST body; mutation_reason required (FR-01, FR-02)
  - `OrgUnitPatch`       - PATCH body; all fields Optional except mutation_reason (FR-08, FR-09)
  - `OrgUnitOut`         - response model for single record and list items
  - `OrgUnitHistoryOut`  - response model for history items (adds replaced_at)
  - `OrgUnitNode`        - response model for tree nodes (adds children: list[OrgUnitNode])
- Dependencies: pydantic

### models/person.py
- Responsibility: define Pydantic schemas for Person.
- Public interface: PersonCreate, PersonPatch, PersonOut, PersonHistoryOut
- Dependencies: pydantic

### models/assignment.py
- Responsibility: define Pydantic schemas for Assignment.
- Public interface: AssignmentCreate, AssignmentPatch, AssignmentOut, AssignmentHistoryOut
- Dependencies: pydantic

### static/index.html
- Responsibility: deliver the entire web GUI as a single self-contained HTML file
  with inlined CSS and JS (FR-33..FR-38).
- Key JS responsibilities:
  - fetch /api/v1/org-units/tree -> render collapsible tree (FR-34)
  - fetch /api/v1/persons        -> render person list (FR-34)
  - on tree node click: fetch /api/v1/org-units/{id}/assignments -> render panel (FR-35)
  - language toggle: swap label strings from a DE/EN dictionary object (FR-36)
  - add/edit/end forms: POST or PATCH with mutation_reason field (FR-37)
  - show inactive toggle: re-fetch with active_only=false (FR-38)
- Dependencies: none (no CDN, no build step)

## Data Flow

### Create OrgUnit (FR-01, FR-02)

```
Browser POST /api/v1/org-units {name, ..., mutation_reason}
  -> FastAPI validates OrgUnitCreate (Pydantic)
     missing mutation_reason -> HTTP 422 (no DB touch)
  -> get_db() yields connection
  -> INSERT INTO org_unit (...) VALUES (...)
     valid_until defaults to "9999-12-31" if omitted (FR-31)
     updated_at = utcnow_str()
  -> return HTTP 201 OrgUnitOut
```

### Update OrgUnit (FR-08, FR-09, FR-32)

```
Browser PATCH /api/v1/org-units/5 {name: "Big Cats", mutation_reason: "Corrected"}
  -> FastAPI validates OrgUnitPatch (Pydantic)
     missing mutation_reason -> HTTP 422 (no DB touch)
  -> get_db() yields connection
  -> copy_to_history(conn, "org_unit", 5, utcnow_str())
     INSERT INTO org_unit_history SELECT *, :now FROM org_unit WHERE id=5
  -> UPDATE org_unit SET name=?, updated_at=?, mutation_reason=? WHERE id=5
  -> return HTTP 200 OrgUnitOut
```

### Build Tree (FR-05)

```
Browser GET /api/v1/org-units/tree
  -> SELECT * FROM org_unit WHERE valid_from<=today AND valid_until>=today
  -> build_tree(rows)         <- pure function, no DB
     group rows by parent_id
     recursively attach children to parents
     return roots (parent_id IS NULL)
  -> return HTTP 200 list[OrgUnitNode]
```

### Active Filter (FR-03, FR-13, FR-22)

```
active_filter(active_only=True, today="2026-05-16")
  -> "WHERE valid_from <= '2026-05-16' AND valid_until >= '2026-05-16'"

active_filter(active_only=False, today=...)
  -> ""   (no filter, all rows returned)
```

## Technology Choices

| Decision | Choice | Rationale |
|---|---|---|
| Web framework | FastAPI + Uvicorn | Auto-generates OpenAPI 3.1.0 (NFR-09); async-native |
| Database driver | aiosqlite | Async SQLite; zero extra containers (NFR-02) |
| ORM | None (raw SQL) | Keeps queries visible; no mapping layer to debug |
| Schema validation | Pydantic v2 (FastAPI built-in) | mutation_reason required enforced at model layer (FR-02, FR-09, etc.) |
| Frontend | Vanilla JS, single HTML file | No build step (FR-33); no CDN dependency |
| Testing | pytest + pytest-asyncio + httpx | AsyncClient against in-memory DB (NFR-01) |
| Dependency mgmt | uv + pyproject.toml | Supply chain rules (NFR-03) |
| Linting | ruff | Single tool for lint + format |
| Container | Podman, python:3.13-slim | NFR-02 |
| Python version | 3.13 | Standing CI rule |

## Database Schema

### Productive tables (same pattern for all 3)

```sql
CREATE TABLE IF NOT EXISTS org_unit (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT    NOT NULL,
    description     TEXT,
    parent_id       INTEGER REFERENCES org_unit(id),
    sort_order      INTEGER NOT NULL DEFAULT 0,
    valid_from      TEXT    NOT NULL,           -- DATE as ISO string
    valid_until     TEXT    NOT NULL DEFAULT '9999-12-31',
    updated_at      TEXT    NOT NULL,           -- DATETIME as ISO string
    mutation_reason TEXT    NOT NULL
);

CREATE TABLE IF NOT EXISTS person (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    first_name      TEXT    NOT NULL,
    last_name       TEXT    NOT NULL,
    email           TEXT,
    notes           TEXT,
    valid_from      TEXT    NOT NULL,
    valid_until     TEXT    NOT NULL DEFAULT '9999-12-31',
    updated_at      TEXT    NOT NULL,
    mutation_reason TEXT    NOT NULL
);

CREATE TABLE IF NOT EXISTS assignment (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    org_unit_id     INTEGER NOT NULL REFERENCES org_unit(id),
    person_id       INTEGER          REFERENCES person(id),  -- NULL = open/unfilled position
    role_label      TEXT,
    valid_from      TEXT    NOT NULL,
    valid_until     TEXT    NOT NULL DEFAULT '9999-12-31',
    updated_at      TEXT    NOT NULL,
    mutation_reason TEXT    NOT NULL
);
```

### History tables (mirror + replaced_at)

```sql
CREATE TABLE IF NOT EXISTS org_unit_history (
    id              INTEGER,
    name            TEXT    NOT NULL,
    description     TEXT,
    parent_id       INTEGER,
    sort_order      INTEGER NOT NULL DEFAULT 0,
    valid_from      TEXT    NOT NULL,
    valid_until     TEXT    NOT NULL,
    updated_at      TEXT    NOT NULL,
    mutation_reason TEXT    NOT NULL,
    replaced_at     TEXT    NOT NULL            -- DATETIME, set by copy_to_history()
);
-- person_history and assignment_history follow the same pattern
-- assignment_history.person_id is also nullable (mirrors productive table)
```

## Testability Constraints

- `get_db()` dependency injection - tests override `app.dependency_overrides[get_db]`
  with a fixture that yields an in-memory aiosqlite connection; no file system
  touch in tests (NFR-01).

- `utcnow_str()` isolation - all timestamp generation goes through this single
  function in database.py; tests monkeypatch it to return a fixed value for
  deterministic assertions.

- `build_tree(rows)` pure function - takes a plain list of dicts, returns a
  nested structure; unit-testable with no DB fixture needed.

- `active_filter(active_only, today)` pure function - returns an SQL fragment
  string; unit-testable with no DB fixture needed.

- `copy_to_history(conn, table, row_id, now)` - accepts an injected connection;
  testable with the in-memory DB fixture.

- Pydantic model validation - `OrgUnitCreate`, `PersonCreate`, `AssignmentCreate`
  and their Patch variants enforce `mutation_reason` as a required non-empty field;
  FastAPI returns HTTP 422 before any endpoint logic runs; testable without DB.

- No global state - DB path is passed into `init_db(path)`; app is created via
  `create_app()` factory; tests instantiate a fresh app per test session.

## Open Questions

1. SQLite TEXT for dates vs. SQLite DATE type - storing as ISO TEXT strings
   (YYYY-MM-DD) is portable and sortable; the application layer compares them
   as strings. Acceptable for v1 given the small data volume. If query performance
   becomes a concern in v2, add indexes on valid_from and valid_until.

2. Concurrency - aiosqlite serialises writes; no concurrent write contention
   expected for a single-zoo deployment. If multi-user simultaneous edits become
   a concern, revisit with WAL mode or PostgreSQL.

3. FR-21 FK validation - org_unit_id is always checked (required). person_id is
   checked only when provided and non-null; NULL person_id is valid and means
   the position is open/unfilled. The explicit SELECT approach is chosen to
   return a meaningful HTTP 422 (not a generic 500).
